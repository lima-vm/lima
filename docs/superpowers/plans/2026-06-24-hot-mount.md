# Hot-Mount Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `limactl mount add|remove|list` to mount/unmount host folders into a running QEMU/Linux VM with no restart, using virtiofs (high-IO) as the primary transport, plus 9p and reverse-sshfs.

**Architecture:** A new `limactl mount` cobra command calls new `/v1/mounts` HTTP endpoints on the hostagent's `ha.sock`. The hostagent keeps a live mount registry. reverse-sshfs reuses the existing host-side `setupMount`. 9p/virtiofs go through an optional `driver.FSHotPlugger` capability implemented only by the QEMU driver (QMP device hot-plug, virtiofsd lifecycle, PCIe-slot allocation), followed by a guest-side `mount` over the existing SSH connection.

**Tech Stack:** Go 1.25, cobra, `digitalocean/go-qemu/qmp` (QMP), `lima-vm/sshocker/pkg/ssh`, net/http over unix socket.

## Global Constraints

- Go module: `github.com/lima-vm/lima/v2`, Go 1.25.7. Build/test via wrapper `/tmp/gow` (sets GOROOT/GOCACHE/GOTMPDIR on the exec-capable `/tmp`).

  ```go
  case runtime.GOOS == "linux":
      // Always back guest memory with a shareable memfd so virtiofs devices can be

      // /dev/shm sizing pitfall of memory-backend-file.
      args = appendArgsIfNoConflict(args, "-object",

      args = appendArgsIfNoConflict(args, "-object",
          fmt.Sprintf("memory-backend-file,id=virtiofs-shm,size=%s,mem-path=/dev/shm,share=on", strconv.Itoa(int(memBytes))))
      args = appendArgsIfNoConflict(args, "-numa", "node,memdev=virtiofs-shm")
  }
  ```

  ```go
  func (d *ConfiguredDriver) HotPlugFS(ctx context.Context, req *HotPlugFSRequest) (*HotPlugFSResponse, error) {
  ```

## Task 4: QEMU `FSHotPlugger` implementation (`pkg/driver/qemu/hotplug.go`)

**Files:**
- Create: `pkg/driver/qemu/hotplug.go`
- Test: `pkg/driver/qemu/hotplug_test.go`
- Modify: `pkg/driver/qemu/qemu_driver.go` (add `hotPlugState` field to `LimaQemuDriver`)

**Interfaces:**
- Consumes: `HotPlugRootPorts`, `HotPlugRootPortID` (Task 2); `driver.HotPlugFSRequest/Response` (Task 3); `mountutil` (Task 1).
- Produces: `*LimaQemuDriver` implements `driver.FSHotPlugger`.

Design notes:
- A `slotAllocator` tracks which of the 8 `lima-hp-N` ports are free (mutex + bool array). `allocate() (id string, ok)`, `free(id)`.
- `virtiofsd` discovered via existing `FindVirtiofsd(ctx, qExe)`; spawned `--socket-path <instanceDir>/<sock> --shared-dir <hostPath>` (+ `--readonly` if not writable and supported); PID tracked. Socket name: `filenames` style under `Instance.Dir`, e.g. `virtiofsd-hp-<slot>.sock`.
- QMP: connect like `changeVNCPassword` (`qmp.NewSocketMonitor`/`Connect`/`raw.NewMonitor`). Issue raw QMP via `qmpClient.Run([]byte)`:
  - virtiofs: `chardev-add` (socket, server=false, path=sock) → `device_add vhost-user-fs-pci,id=<dev>,chardev=<char>,tag=<tag>,bus=<port>,queue-size=<n>`.
  - 9p: `human-monitor-command` `fsdev_add local,id=<fsdev>,path=<hostPath>,security_model=none[,readonly=on]` → `device_add virtio-9p-pci,id=<dev>,fsdev=<fsdev>,mount_tag=<tag>,bus=<port>`.
  - remove: `device_del <dev>` → wait `DEVICE_DELETED` (subscribe via monitor `Events`, timeout 30s) → `chardev-remove`/`fsdev_del` → free slot, kill virtiofsd.
- `DeviceID` encodes the slot + device id so `HotUnplugFS` can reverse without external state, but also keep an in-driver map `deviceID → {slot, char/fsdev, virtiofsdPID}`.

## Task 5: Hostagent mount registry + add/remove/list

**Files:**
- Modify: `pkg/hostagent/mount.go`, `pkg/hostagent/hostagent.go`
- Test: `pkg/hostagent/mount_test.go`

**Interfaces:**
- Consumes: `driver.FSHotPlugger` via `a.driver` (type-assert to `driver.FSHotPlugger`); `mountutil` (Task 1); `ssh.ExecuteScript`.
- Produces (consumed by Task 6 API):
  ```go
  func (a *HostAgent) MountAdd(ctx context.Context, hostPath, guestPath string, mountType limatype.MountType, writable bool) (*hostagentapi.Mount, error)
  func (a *HostAgent) MountRemove(ctx context.Context, guestPath string) error
  func (a *HostAgent) MountList() []hostagentapi.Mount
  ```

Design notes:
- Registry on HostAgent: `hotMounts map[string]*activeMount` keyed by guestPath + `hotMountsMu sync.Mutex`. `activeMount{guestPath, hostPath string; mountType limatype.MountType; writable bool; close func() error}`.
- reverse-sshfs add → build a `limatype.Mount` (fill SSHFS defaults), call existing `setupMount`, store `close`.
- 9p/virtiofs add → assert `driver.FSHotPlugger`; build a `limatype.Mount` to compute `mountutil.Tag` and `mountutil.MountOptions`; `HotPlugFS`; then `ssh.ExecuteScript` runs `sudo mkdir -p <gp> && sudo mount -t <fstype> -o <opts> <tag> <gp>`. On guest-mount failure → `HotUnplugFS` rollback. Store close = `umount over ssh` + `HotUnplugFS`.
- Validation: reject if guestPath already in registry; reject system mountpoints (reuse rule from `pkg/limayaml/validate.go`); reject non-existent host dir; require instance running (checked at CLI layer too).
- Default type: if `mountType == ""`, default `limatype.VIRTIOFS` on Linux+QEMU.
